程式語言可以分為兩種:

編譯式語言(Compiled Language)

例如 Java、C、C#、C++、Pascal
流程 原始碼 → 預處理器 → 代碼生成 → 執行
特性 效能比較好、除錯比較快、程式碼可獨立執行。

直譯式語言(Interpreted Language)

例如 JavaScript、Python、PHP、Ruby、BASIC、LISP、Perl、R
流程 原始碼 → 直譯器 → 代碼生成 → 執行
特性 程式碼由上到下執行，效能比較差、除錯比較慢、需要依賴執行環境。

作用域可分為兩種:

動態作用域(Dynamic Scope)

  • 特性 : 作用域在函式調用時才決定。

靜態作用域(Lexical Scope)

  • 又稱 : 語法作用域。
  • 特性 : 語法在解析時就決定作用域，而且不再改變。

執行環境(Execution Context)

  • 區域環境
    • 在function內，會產生自己的this。
    • 可以重複被執行產生環境。
  • 全域環境
    • 瀏覽器開啟時產生的window。
    • window === this。

執行堆疊(Execution stack)

  • 全域環境最先堆疊，再來依序堆疊區域環境。
  • 區域環境依序結束，最後留下全域執行環境。

範圍鍊(Scope Chain)

  • 當函式本身沒有變數時，會向外層找到全域變數。(範例一)
  • 當函式裡的函式本身沒有變數時，會向外找到外層函式裡的變數。(範例二)

/** 範例一 **/
var a = 1;
function fn1(){
  var a = 2;
  fn2();
}
function fn2(){
  console.log(a);
}
fn1();
//console.log(a);  1
//因為function fn2本身沒有變數，所以會找向外層找全域變數;

/** 範例二 **/
var a = 1;
function fn1(){
  var a = 2;
  function fn2(){
    console.log(a);  //2
  }
  fn2();
}
fn1();
//因為函式(fn2)本身沒有變數，所以會找向外層找函式(fn1)的變數;

提升(Hoisting)

  • 在創造階段時，宣告變數、函式表達式會存放在記憶體裡，還取不到值(undefined)。(範例一)
  • 函式陳述式會優先執行。(範例二)

多執行緒(Multi thread)

  • 一次能完成很多事

單執行緒(Single Thread)

  • 同步事件會依序執行。
  • 非同步事件會先移到 事件佇列(Event Queue) ，等待其他事件執行完才會執行。
    • setTimeout(範例一)
    • addEventListener(範例二)
